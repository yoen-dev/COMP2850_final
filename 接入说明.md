# NourishWell 前后端接入说明
生成时间：2026-04-20 | 后端：Kotlin + Spring Boot，http://localhost:8080

---

## 文件清单

| 文件 | 作用 |
|------|------|
| `api.js` | 所有 API 调用的统一封装层，所有页面必须先加载 |
| `patch_index.js` | 覆盖 index.html 的 handleLogin / handleRegister |
| `patch_dashboard.js` | 覆盖 dashboard.html 的日记/运动/食谱/收藏/评分/评论 |
| `patch_pro_dashboard.js` | 覆盖 pro_dashboard.html 的 clients/消息/预约 |

---

## 第一步：将 4 个 js 文件放到项目根目录

```
COMP2850_final/
  api.js                  ← 新增
  patch_index.js          ← 新增
  patch_dashboard.js      ← 新增
  patch_pro_dashboard.js  ← 新增
  index.html
  dashboard.html
  pro_dashboard.html
  qa_subscriber.html      ← 无需改动
  qa_pro.html             ← 无需改动
```

---

## 第二步：在 index.html 里加两行

找到 `</body>` 标签（文件最末尾），在它**前面**插入：

```html
  <!-- API Integration -->
  <script src="api.js"></script>
  <script src="patch_index.js"></script>
</body>
```

**效果：**
- 登录成功 → token 存入 sessionStorage → 根据 role 跳转
- 注册成功 → 同上
- 邮箱/密码错误 → 显示对应错误信息（原来是 setTimeout 假跳转）

---

## 第三步：在 dashboard.html 里加两行

同样找到 `</body>` 前插入：

```html
  <!-- API Integration -->
  <script src="api.js"></script>
  <script src="patch_dashboard.js"></script>
</body>
```

**效果：**
- 未登录访问 → 自动跳回 index.html
- 页面加载 → 从 /api/recipes 拉取食谱，/api/favourites 初始化收藏，/api/diary 加载今日日记
- 切换日期 → 从 API 拉取对应日期数据
- 添加食物 → POST /api/diary，保留后端 id 用于删除
- 删除食物 → DELETE /api/diary/:id
- 添加运动 → POST /api/exercise
- 删除运动 → DELETE /api/exercise/:id
- 收藏/取消收藏 → POST/DELETE /api/favourites
- 提交评分 → POST /api/ratings（upsert）
- 提交评论 → POST /api/comments
- Logout → 清除 token，跳回首页

---

## 第四步：在 pro_dashboard.html 里加两行

```html
  <!-- API Integration -->
  <script src="api.js"></script>
  <script src="patch_pro_dashboard.js"></script>
</body>
```

**效果：**
- 未登录/非专业人士访问 → 自动跳转
- 页面加载 → 从 /api/clients 拉取客户列表，映射到静态数据 key
- 点击客户 → 加载该客户消息、预约、近 30 天日记摘要
- 发消息 → POST /api/messages
- 预约 → POST /api/appointments
- 切换预约状态 → PATCH /api/appointments/:id

## 降级机制（后端未启动时）

所有 patch 里的 API 调用都包裹在 try/catch 里：
- **食谱**：API 失败 → 继续用 HTML 里的静态 RECIPES 对象
- **日记/运动**：API 失败 → 显示空状态，不崩溃
- **收藏**：API 失败 → 乐观更新回滚，UI 恢复原状
- **消息/预约**：API 失败 → toast 提示，UI 乐观更新回滚

这意味着演示时后端没跑也不会白屏，只是数据不持久化。

## 已知后端限制（文档里写明的）

| 限制 | 前端处理方式 |
|------|-------------|
| diary 里 carbs/fat/time 是占位值 | 前端直接展示，不做断言 |
| comments/ratings 没有 DELETE 接口 | 前端不展示删除按钮 |
| 测试/CI 由另一位组员补充 | 不影响前端 |

## 如果后端跑在非 8080 端口

修改 `api.js` 第 6 行：

```js
const BASE = 'http://localhost:8080';  // 改成实际端口
```

## 快速排错

| 现象 | 原因 | 解决 |
|------|------|------|
| 登录后跳回 index | token 未存入 | 检查后端是否返回 token 字段 |
| 日记加载为空 | 日期格式不对 | 确认后端接受 YYYY-MM-DD |
| 收藏失败 | recipe 没有 _numericId | 先确保 /api/recipes 已调通 |
| 专业人士页面看不到客户 | clients name 不匹配 | 检查后端 name 字段与前端 clients 对象的 name 是否一致 |
| 401 错误 | token 过期 | 重新登录 |
